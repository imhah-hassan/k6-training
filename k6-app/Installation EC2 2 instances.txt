ou
# 1. Créez le dossier système pour les plugins Docker
sudo mkdir -p /usr/libexec/docker/cli-plugins

# 2. Téléchargez la toute dernière version stable de Docker Compose
sudo curl -SL https://github.com/docker/compose/releases/latest/download/docker-compose-linux-$(uname -m) -o /usr/libexec/docker/cli-plugins/docker-compose

# 3. Donnez les droits d'exécution au fichier téléchargé
sudo chmod +x /usr/libexec/docker/cli-plugins/docker-compose

1. Les Dashboards adaptés à InfluxDB (v1.8)
Pour visualiser vos conteneurs Docker et votre instance EC2 avec les données envoyées par Telegraf dans InfluxDB, utilisez plutôt ces IDs :

ID 10585 (Docker Dashboard via Telegraf) : C'est le plus complet pour voir le CPU, la RAM et le réseau de chaque conteneur (OrangeHRM, MariaDB, etc.).
Varaiable DS_INFLUXDB=influxdb
ID 1150 (Telegraf: System Dashboard) : Très efficace pour surveiller l'état global de votre machine hôte (CPU total, Load Average, Disk I/O).
Varaiable DS_INFLUX=influxdb

. Les meilleurs Dashboards Grafana pour k6 + InfluxDB
ID 2587 (k6 Load Test Results) : C'est le dashboard "officiel" et le plus utilisé pour k6 avec InfluxDB. Il est simple, efficace et affiche les RPS (Requêtes par seconde), les erreurs et les percentiles de latence (P90, P95, P99).
ID 10660 (k6 Cloud Style) : Si vous préférez une interface plus moderne qui ressemble à l'interface k6 Cloud, celui-ci est excellent. Il nécessite que k6 envoie ses données dans une base InfluxDB nommée k6 (ou de changer la variable dans le dashboard).
